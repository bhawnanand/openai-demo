How it works

ShopScript calls OpenAI's API in real time. If the API is unavailable, it automatically falls back to local generation so the demo keeps working.

Live API active
🟢 Live API Mode

OpenAI generates your product page in real time with custom benefits, FAQs, and SEO metadata based on your brief.

🟡 Fallback Mode

If the API is slow or unavailable, a local generator creates a valid page so the demo keeps working.

Production Ready

The same API boundary scales to real authentication, databases, image generation, and approval workflows.

// What you send:
POST /api/generate {
  name: "Product name",
  description: "What it is",
  price: "$X",
  audience: "Who it's for",
  tone: "Style (e.g., premium)"
}

// What you get back:
{
  headline: "AI-generated headline",
  benefits: ["...", "...", "..."],
  faqs: [{question, answer}, ...],
  seo: {title, description},
  visualSvg: ""
}

Note: The API key is kept on the server (not in browser) for security. In production, add real auth, database persistence, rate limiting, and observability.
